**Passe 1** — accès RPC réel obtenu (feu vert opérateur explicite), lecture directe

vrai token en bonding). Réserve initiale du token agent confirmée = exactement

**Passe 2** (sur demande opérateur "pousse plus loin") — vrai contrat trouvé par
balayage direct des logs on-chain (`eth_getLogs` sur `Graduated(address,address)`, par
blocs de 10 000 à cause de la limite du RPC public) : `0x1A540088125d00dD3990f9dA45CA0859af4d3B01`
(proxy EIP-1967 vérifié Blockscout). Cause racine du seuil global faux trouvée en lisant
`launchpadv2/BondingV5.sol` du repo `Virtual-Protocol/protocol-contracts` (PAS
`fun/Bonding.sol`, lu en passe 1 — deux générations de contrats coexistent dans le même
repo) : le seuil n'est plus une constante globale mais **par token**
(`tokenGradThreshold(address)`). Formule validée empiriquement sur un vrai token gradué :
`reserve0 = 163 037 405 <= tokenGradThreshold = 168 316 832` — exactement la condition
de graduation du code source.

**Précision nomenclature demandée par l'opérateur** : une entrée `CLAUDE.md` antérieure
avait été rappelée comme mentionnant "BONDING_V4" comme version la plus récente connue.

`HANDOFF-*.md`, tout le code, tout `knowledge/*.yaml`) : **aucune occurrence de
"BONDING_V4" trouvée nulle part dans ce dépôt.** Les seules mentions "V4" présentes
concernent **Clanker v4** (un launchpad différent, adresses de deployer documentées dans

utilisée par les pools Zora/Clanker) — sujets voisins dans la même doc de recherche, mais
